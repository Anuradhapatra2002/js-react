DOM Events

- DOM events are somethings that happen on a web page which our code can notice.

- These can be caused by the user (by clicking, typing, or hovering) or by the browser.

- There are several types of DOM Events eg:

  • Mouse Events
  • Keyboard Events
  • Form Events
  • Document Events
  • Drag & Drop Events etc.

DOM Event Listeners

- A DOM event listener is a function that waits for an event to happen and then reacts to it.

Ex:-

- Events are like signals: something just happened.

- Listeners are like peoples: they notice the event and do something in response.

- There are 3 ways in which we can add an event listener

  • by using Inline HTML attributes
    • Only one handler per event type
  • by using event properties (like onclick)
    • Only one handler per event type
    • Overwriting occurs if you assign again
  • by using 'addEventListener'
    • Multiple listeners allowed

Event Object

- In JavaScript, an Event Object is a special object that is automatically passed to event handler functions when an event occurs.

- It contains information about the event and provides methods and properties to interact with it.

- Key properties & methods

  • type
  • target
  • preventDefault()
  • stopPropagation()
  • clientX, clientY
  • key, code, keycode
